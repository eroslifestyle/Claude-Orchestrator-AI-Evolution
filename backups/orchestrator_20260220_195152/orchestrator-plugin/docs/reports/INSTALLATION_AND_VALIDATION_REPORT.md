## 🎯 AVAILABLE COMMANDS

The Claude Code Orchestrator Plugin provides these production-ready commands:

### 1. `/orchestrator <task>`
**Main orchestration command** - Analyzes task and coordinates multiple expert agents
- **Parameters:** task (required), preview (boolean), parallel (1-64), model (auto/haiku/sonnet/opus)
- **Functionality:** Complete task analysis, agent coordination, parallel execution
- **Example:** `/orchestrator "Create authentication system with JWT and OAuth2"`

### 2. `/orchestrator-preview <task>`
**Preview orchestration plan** without execution
- **Parameters:** task (required)
- **Functionality:** Shows agent coordination plan, estimated duration, complexity analysis
- **Example:** `/orchestrator-preview "Build RESTful API with database"`

### 3. `/orchestrator-status`
**Show current orchestration status** and system metrics
- **Parameters:** None
- **Functionality:** Plugin status, capabilities, performance metrics, expert agent count
- **Example:** `/orchestrator-status`

### 4. `/orchestrator-agents`
**List available expert agents** and their specializations
- **Parameters:** None
- **Functionality:** Complete expert registry, specializations, keywords, success rates
- **Example:** `/orchestrator-agents`

### 5. `/orchestrator-benchmark [agents] [duration]`
**Run performance benchmarks** for parallel system
- **Parameters:** agents (1-64, default 12), duration (seconds, default 300)
- **Functionality:** Stress testing, performance validation, scalability analysis
- **Example:** `/orchestrator-benchmark 32 600`

### 6. `/orchestrator-config [setting] [value]`
**Configure orchestrator settings**
- **Parameters:** setting (optional), value (optional)
- **Functionality:** System configuration, model preferences, parallelism settings
- **Example:** `/orchestrator-config maxParallelism 64`
